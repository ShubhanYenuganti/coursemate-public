# Server-Funded Free Tier Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give signed-in users with no API key a daily allowance of chat messages on a server-funded cheap model, with a visible counter and an upgrade prompt at the cap.

**Architecture:** Pure `free_tier_decision` budget function + `free_tier_usage` table; the chat send path resolves credentials (own key vs server free-tier key + cheap model) and increments usage on success; the composer shows remaining quota; landing copy changes.

**Tech Stack:** Python serverless, Neon Postgres, pytest, React.

**Spec:** `docs/superpowers/specs/2026-06-10-free-tier-design.md`

# api/services/free_tier.py
FREE_TIER_DAILY_MESSAGES = 10
FREE_TIER_PROVIDER = "gemini"
FREE_TIER_MODEL_ID = "gemini-3-flash-preview"

def free_tier_decision(used_today: int, has_own_key: bool, cap: int = FREE_TIER_DAILY_MESSAGES) -> dict:
    if has_own_key:
        return {"allowed": True, "remaining": None, "reason": "own_key", "use_free_tier": False}
    remaining = max(0, cap - used_today)
    if remaining <= 0:
        return {"allowed": False, "remaining": 0, "reason": "free_tier_exhausted", "use_free_tier": True}
    return {"allowed": True, "remaining": remaining, "reason": "free_tier", "use_free_tier": True}
```

def used_today_sql() -> str:
    return "SELECT count FROM free_tier_usage WHERE user_id = %s AND usage_date = CURRENT_DATE"

def increment_usage_sql() -> str:
    return (
        "INSERT INTO free_tier_usage (user_id, usage_date, count) "
        "VALUES (%s, CURRENT_DATE, 1) "
        "ON CONFLICT (user_id, usage_date) DO UPDATE SET count = free_tier_usage.count + 1 "
        "RETURNING count"
    )
```

- [ ] **Step 4: Implement the resolver and integrate**

In `api/chat.py`:

```python
from .services.free_tier import free_tier_decision, FREE_TIER_PROVIDER, FREE_TIER_MODEL_ID  # + import branch

def resolve_send_credentials(user_id, requested_provider, has_own_key, used_today):
    decision = free_tier_decision(used_today, has_own_key)
    if decision["use_free_tier"]:
        return True, FREE_TIER_PROVIDER, FREE_TIER_MODEL_ID, decision
    return False, requested_provider, None, decision
```

In `do_POST`, before generating:
- compute `has_own_key` (any provider key present) and `used_today` (run `used_today_sql()`),
- call `resolve_send_credentials(...)`,
- if `not decision["allowed"]`: `send_json(self, 429, {"error": "free_tier_exhausted", "message": "You've used today's free messages — add an API key in Profile for unlimited chat."})` and return,
- if `use_free_tier`: use the server free-tier key (read from env, e.g. `os.environ["FREE_TIER_GEMINI_KEY"]`) and `FREE_TIER_MODEL_ID` when calling the model instead of the user key path,
- after a successful generation, run `increment_usage_sql()` when `use_free_tier`.
- include `free_remaining` in the response payload so the UI can show the counter.
